## 방향성 전제

> **"텍스처 파이프라인은 처음부터 올바르게. 폴리곤 최적화는 형태가 완성된 후."**

## 전체 로드맵

```
Phase 1  │ 기초 구조        │ 1~2주   │ 이론 50 : 실습 50
Phase 2  │ 잎과 텍스처      │ 3~5주   │ 이론 30 : 실습 70
Phase 3  │ 실사 나무        │ 6~9주   │ 이론 20 : 실습 80
Phase 4  │ 특수 식생        │ 10~12주 │ 이론 20 : 실습 80

Phase 6  │ 고급 기능        │ 15주~   │ 이론 30 : 실습 70
```

## Phase 1 — 기초 구조 이해 (1~2주, 14일)

**목표:** 노드 트리가 어떻게 동작하는지 원리를 이해하고, 파라미터 하나를 바꿨을 때 왜 그런 결과가 나오는지 설명할 수 있는 수준

### Week 1 — 노드 트리와 제너레이터 개념

|일차|유형|내용|
|---|---|---|
|Day 1|이론|노드 트리 계층 구조 이해 — Root → Trunk → Branch → Leaf 관계|
|Day 2|이론|Trunk 제너레이터 집중 — Length, Radius, Segments, Flares 파라미터|
|Day 3|실습|Trunk만으로 다양한 줄기 형태 만들기 (굵은/가는/구부러진)|
|Day 4|이론|Branch 제너레이터 — Frequency, Angle, Spread, Length 파라미터|
|Day 5|실습|Trunk + Branch 1레벨로 단순 나뭇가지 구조 완성|
|Day 6|이론|Branch Level 2 추가 — 계층이 깊어질 때 파라미터 상속 개념 이해|
|Day 7|실습|Trunk → Branch L1 → Branch L2 구조 자유롭게 실험|

### Week 2 — Leaf와 첫 결과물

|일차|유형|내용|
|---|---|---|
|Day 8|이론|Leaf 제너레이터 타입 개요 — Plane, Frond, Mesh 차이|
|Day 9|실습|Plane 타입 잎 붙이기 — Size, Frequency, Distribution 조정|
|Day 10|이론|Gravity, Noise 파라미터 — 잎에 자연스러운 무게감 주기|
|Day 11|실습|기본 텍스처(SpeedTree 기본 제공 리소스) 연결해보기|
|Day 12|이론|Seed 개념 — 프로시저럴 랜덤성 이해, Variation 관리 방법|
|Day 13|실습|Phase 1 결과물 제작 — 단순한 풀꽃 1개 완성|
|Day 14|복습|만든 것 분석 — 어떤 파라미터가 어떤 형태를 만들었는지 정리|

**Phase 1 완료 기준:** 노드 트리를 보고 "이 제너레이터는 이런 역할"이라고 즉시 설명할 수 있다.

## Phase 2 — 잎과 텍스처 파이프라인 (3~5주, 21일)

**목표:** 텍스처를 올바른 채널 구조로 만들고, 실사 수준의 잎 머티리얼을 SpeedTree 안에서 구성할 수 있는 수준

> 이 Phase가 전체 커리큘럼에서 가장 중요합니다.  
> 텍스처 파이프라인을 여기서 정확히 잡으면 이후 모든 식생에 재활용됩니다.

## 학습 시 일반 원칙

- **레퍼런스를 항상 옆에 둔다.** 식물 사진을 보면서 "이게 어떤 파라미터로 표현되는가"를 분석하는 습관
- **망가뜨려라.** 파라미터를 극단적으로 올리거나 내려서 한계를 파악하는 것이 이론보다 빠름
- **하루 실습 결과를 4_render 폴더에 스크린샷으로 저장한다.** 성장 추적용
- **막히면 단순화한다.** 복잡한 형태가 안 될 때는 제너레이터를 하나씩 지워보면서 문제를 찾음
- **SpeedTree 기본 샘플 파일을 분해한다.** Browse samples에서 비슷한 식생을 열어서 노드 트리를 분석하는 것이 가장 빠른 학습법
