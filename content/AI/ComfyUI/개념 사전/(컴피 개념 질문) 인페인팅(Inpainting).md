## 한 줄 요약
> 이미지의 특정 영역만 마스크로 지정해서 그 부분만 새로 생성/수정하는 기법.

## 비유로 이해하기

벽에 그림이 걸려 있는데, 일부분에 구멍이 났다고 상상해보자. 인페인팅은 그 구멍 부분만 새로 그려 넣되, 주변 그림과 자연스럽게 이어지도록 채우는 작업이다. 나머지 벽화는 그대로 두고, 손상된 부분만 복원하는 미술 복원사와 같은 역할.

---
## 인페인팅(Inpainting)이 하는 일

이미지 전체를 다시 생성하는 것이 아니라, 마스크로 칠한 영역만 프롬프트에 따라 새로 생성한다. 마스크 바깥 영역은 원본 그대로 유지되고, 마스크 안쪽만 주변 맥락에 맞게 채워진다.

예를 들어:
- 배경의 불필요한 오브젝트 제거 (사람, 간판 등)
- 캐릭터의 옷이나 소품만 교체
- 텍스처의 특정 부분만 수정
- 이미지 일부가 마음에 안 들 때 그 부분만 재생성

### AI 모델에서 인페인팅이 왜 중요한가?

txt2img로 이미지를 생성하면 전체가 한 번에 나오기 때문에, 95%는 마음에 드는데 5%만 고치고 싶은 경우가 자주 생긴다. 인페인팅 없이는 전체를 다시 뽑아야 하므로 마음에 들었던 부분까지 바뀌어버린다. 인페인팅은 원하는 부분만 정밀하게 수정할 수 있게 해주는 핵심 편집 기법이다.

### 문제 상황(인페인팅이 없을 때)

- txt2img로 배경 에셋을 생성했는데 한쪽 구석에 이상한 아티팩트가 생김 → 전체를 다시 생성해야 함
- 캐릭터 이미지에서 손 부분만 이상하게 나옴 → 전체를 다시 뽑으면 얼굴까지 바뀜
- 텍스처 이미지에서 특정 영역만 패턴이 어긋남 → 부분 수정 불가

## ComfyUI에서 사용 방법 및 필수 조건

1. **기본 인페인팅 워크플로우 (내장 노드만 사용)**
   - `Load Image` → IMAGE 출력을 `VAE Encode (for Inpainting)` 노드의 pixels 입력에 연결
   - `Load Image`에서 마스크를 직접 그리거나, 별도 마스크 이미지를 mask 입력에 연결
   - `VAE Encode (for Inpainting)` → LATENT → `KSampler` → `VAE Decode` → `Save Image`
   - KSampler의 denoise 값으로 변화 강도를 조절 (0.4~0.7: 원본 느낌 유지, 0.8~1.0: 크게 변경)

2. **마스크 지정 방법**
   - ComfyUI에서 `Load Image` 노드의 이미지를 우클릭 → "Open in MaskEditor"로 직접 마스크를 그릴 수 있음
   - 또는 SAM3 같은 세그멘테이션 노드로 자동 마스크를 생성해서 연결

3. **필수 조건**
   - 체크포인트 모델 (SD 1.5, SDXL 등 일반 모델 사용 가능, 인페인팅 전용 모델도 있음)
   - VAE (체크포인트에 내장되어 있거나 별도 로드)
   - 마스크 (수정할 영역을 흰색으로 칠한 흑백 이미지)

4. **인페인팅 전용 모델 vs 일반 모델**
   - 전용 모델 (예: `sd-v1-5-inpainting.ckpt`)은 마스크 영역의 경계 처리가 더 자연스러움
   - 일반 모델로도 인페인팅은 가능하지만, 경계가 부자연스러울 수 있음
   - SDXL 기반이라면 일반 모델 + `VAE Encode (for Inpainting)` 조합으로도 충분히 좋은 결과가 나옴

5. **VRAM 절약 팁**
   - 인페인팅은 이미지 전체를 latent로 인코딩하므로 VRAM 사용량은 일반 생성과 비슷함
   - 큰 이미지를 인페인팅할 때는 crop 영역만 잘라서 처리하면 VRAM을 아낄 수 있음 (comfyui-impact-pack의 `CropRegion` 활용 가능)

## 참고

- ComfyUI 기본 워크플로우 흐름: `Load Image` + 마스크 → `VAE Encode (for Inpainting)` → `KSampler` → `VAE Decode` → `Save Image`
- 네이밍 규칙에 따른 파일명 예시: `EDIT_img2img_sd15-inpaint_v1`
- SAM3로 마스크를 자동 생성한 뒤 인페인팅에 연결하면, 텍스트로 영역 지정 → 해당 영역만 수정하는 반자동 워크플로우도 구성 가능
