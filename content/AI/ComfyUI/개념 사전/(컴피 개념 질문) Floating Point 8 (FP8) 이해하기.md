## 한 줄 요약

> FP8(Floating Point 8)은 AI 모델의 숫자 데이터를 8비트로 압축 저장하는 방식으로, 품질 손실은 거의 없으면서 용량과 속도를 대폭 줄인 "경량화 버전"이다.

- 8/16/32 등 숫자가 붙은거는 다 이 개념으로 이해
	- 숫자가 2배 커질 때마다 → 용량/메모리도 **딱 2배**씩 늘어남 (Linear)
```
FP8  →  FP16  →  FP32
 x2        x2
```

## 비유로 이해하기

소수점 숫자를 **자릿수 제한이 있는 메모장**에 적는다고 생각해보자.

|형식|자릿수|예시|
|---|---|---|
|FP32 (풀버전)|32칸|3.14159265358979|
|FP16|16칸|3.1415|
|FP8|8칸|3.14|

칸이 적을수록 정밀도는 살짝 떨어지지만, **용량과 속도가 줄어든다.**

---

## FP8이 하는 일

AI 모델을 구성하는 수십억 개의 숫자(가중치)를 **더 적은 비트로 압축**해서 저장한다.  
숫자 표현의 정밀도를 조금 희생하는 대신, 모델을 훨씬 가볍고 빠르게 만든다.

### AI 모델에서 FP8이 왜 중요한가?

Flux Dev 같은 대형 모델은 수십억 개의 숫자로 이루어져 있어서, 저장 방식에 따라 용량이 크게 달라진다.

|버전|용량|
|---|---|
|Flux Dev 원본 (FP32/BF16)|약 24GB 이상|
|Flux Dev FP8|약 12GB 수준|

### 문제 상황 (FP8이 없을 때)

- 풀버전(FP32) 모델은 VRAM을 너무 많이 차지해서 **일반 소비자용 GPU로는 실행 자체가 불가능**
- 모델 로딩 속도도 느리고, 이미지 생성 시간도 길어짐
- 고사양 GPU가 없으면 고품질 모델을 사용하기 어려운 진입장벽 발생

## 사용 방법 및 필수 조건

1. Civitai 또는 Hugging Face에서 모델명에 **fp8**이 붙은 파일을 다운로드
2. ComfyUI의 `models/checkpoints/` 폴더에 넣기
3. 일반 체크포인트와 동일하게 로드해서 사용 (별도 설정 불필요)
4. ComfyUI에서 FP8 모델을 쓰려면 **CUDA 지원 GPU** 필요 (CPU 전용 환경에서는 호환 안 될 수 있음)

## 참고

- FP8 vs 원본의 화질 차이는 대부분의 경우 **육안으로 구별하기 어려운 수준**
- VRAM이 넉넉하다면 FP16이나 BF16이 품질 면에서 조금 더 유리
- FP8은 특히 **VRAM 12GB 이하 환경**에서 Flux 계열 모델을 돌릴 때 사실상 필수
