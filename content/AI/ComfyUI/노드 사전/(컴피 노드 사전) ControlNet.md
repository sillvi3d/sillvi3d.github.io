## ControlNet이란?

이미지 생성 시 **구도, 포즈, 윤곽선** 등을 정밀하게 제어할 수 있도록 도와주는 보조 노드.  
프롬프트만으로는 제어하기 어려운 "형태적 정보"를 별도로 입력해서 생성의 기준으로 삼는다.

## 한 줄 요약

> 이미지 생성의 "스케치 위에 덧그리기" — 형태는 고정한 채로 스타일만 바꿀 수 있게 해주는 도구

### 주요 ControlNet 종류

| 종류            | 추출하는 정보     | 활용 예시                 |
| ------------- | ----------- | --------------------- |
| **OpenPose**  | 사람의 관절 포즈   | 특정 포즈 그대로 재현          |
| ==**Canny**== | ==윤곽선(엣지)== | ==구도·형태 유지하며 스타일 변경== |
| ==**Depth**==     | ==원근감·깊이==      | ==3D 공간감 유지==             |
| **Scribble**  | 대충 그린 스케치   | 낙서를 정식 이미지로 변환        |
| **Lineart**   | 정밀한 선화      | 일러스트 채색               |
| **Seg**       | 구역 분할 정보    | 배경·인물 영역 구분           |

---

## ControlNet이 하는 일

입력 이미지에서 **특정 정보(포즈/윤곽/깊이 등)만 추출**해서, 새로운 이미지 생성 시 그 정보를 기준으로 반영한다.

```
[원본 이미지]
     │
     ▼
[ControlNet 전처리기]  ← 포즈/윤곽선/깊이 등 추출
     │
     ▼
[ControlNet 노드]  ← 추출한 정보를 생성에 반영
     │
     ▼
[새로운 이미지]  ← 구도/포즈가 원본과 동일하게 유지됨
```

### 문제 상황 (ControlNet이 없을 때)

- "두 손을 들고 있는 사람"을 프롬프트로만 생성하면 손 위치가 **매번 랜덤**하게 달라짐
- 원하는 구도나 포즈를 **재현하는 것이 사실상 불가능**
- 스타일만 바꾸고 싶어도 구도까지 함께 바뀌어버림

---

## 사용 방법 및 필수 조건

1. **전처리기(Preprocessor)** 노드로 원본 이미지에서 원하는 정보(포즈, 엣지 등) 추출
2. 추출된 정보를 **ControlNet 로더** 노드에 연결
3. ControlNet 노드의 출력을 **KSampler의 conditioning 입력**에 연결
4. 용도에 맞는 **ControlNet 전용 모델 파일** 필요 (Civitai 또는 Hugging Face에서 다운로드)
5. 베이스 체크포인트와 **아키텍처가 일치**해야 함 (SD1.5용 / SDXL용 / Flux용 구분)

## 참고

- 전처리기와 ControlNet 모델은 **세트로 맞춰서** 써야 함 (예: Canny 전처리기 → Canny ControlNet 모델)
- `strength` 값으로 ControlNet의 영향력 조절 가능 (0~1, 보통 0.6~0.9 권장)
- 여러 ControlNet을 **동시에 중첩**해서 사용하는 것도 가능 (포즈 + 깊이 동시 적용 등)
- Flux 전용 ControlNet은 아직 SD 계열보다 생태계가 작으므로 주의
