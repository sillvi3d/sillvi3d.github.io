1. 옵시디언 동기화 방식 변경
	- 기존: remotely save(onedrive) 방식 사용
		- 동기화 불안전성. 파일이 날아가는 일 종종 있었음.
		- 최종본에 계속 덮어쓰는 구조
		- pc, 안드로이드, mac 모두 무료로 동기화 가능
	- 변경: Obsidian git 사용
		- git repo 기반으로 버전 관리 가능
		- git 플러그인으로 pc간 동기화는 쉬움
		- 안드로이드는 termux 사용해서 깃 클론
			- 1gb 넘어가면 staus 확인하는데 시간 소요 오래 걸림
		- mac은 working copy라는 유료 앱 사용 필요
		- 일단 나는 pc-안드로이드만 연결해서 사용할 예정
	- 덩달아 onedrive를 기반으로 두었던 vault를 repo로 바꾸었음

2. git site 개설
	- 나는 옵시디언에서 글을 쓰고 -> 바로 블로그에 올리는 시나리오를 생각함.
		- 옵시디언의 마크다운 공식으로 네이버 블로그에서 붙여넣었을 때 `##`<-마크다운 기호가 다 노출되는 문제. 일일이 수정 필요
	- 옵시디언과 연동이 가장 좋은 quartz 사용
		- quartz 전용 레포를 만들고 그것이 곧 블로그가 됨.
			- 클론을 받으면 css나 스타일 바꿀 수 있게 파일들이 있음
			- 내 옵시디언 볼트의 특정 폴더를 블로그로 사용하도록 경로 설정.
			- publish하는 파이선 코드 작성
	- 개인 vault와 blog 업로드용 폴더를 겸용으로 사용하기 때문에 이미지를 관리할 폴더를 개인 딴에서 폴더를 만들어 관리. 모든 인풋 이미지는 해당 폴더에 있어야만 인식되도록 설정
		- 또한 |200 같이 사이즈를 조정한 경우도 처리할 수 있도록 코드 수정

3. publish.py를 터미널이 아닌 옵시디언 내부에서 실행시킬 수 있도록 shortcut 제작
	- shell commend 플러그인 사용해 연결
	- 단축키: ctrl shift p(publish 약자)

4. publish 성공하면 윈도우 알람 오도록 설정
	- winotify 사용

5. CSS 건들여보기
	- 슈퍼미트보이 느낌으로 바꿔보려 했으나 건들면 건들수록 가독성이 떨어져 우선은 기본값 사용하기로 했음.
	- css는 quartz > quartz > style > custom 파일 수정
	- 메인에 뜨는 메세지와 크기(마크다운 사용) quartz > index 파일 수정

6. json 파일 옵시디언 내부에 인식할 수 있도록 설정
	- 기본적으로 옵시디언은 내부 인풋으로 받을 수 있는 확장자를 제한해 놓음
		- 설정 > 파일 및 링크 > 모든 확장자 인식 체크해주면 json인식됨

7. 블로그에 댓글창, 방문자 통계 연동
	- 댓글창
		- Giscus 사용. GitHub Discussions 기반이라 GitHub 계정으로 댓글 달 수 있고, Quartz랑 궁합이 좋다고 한다.
		- 사용하려면 활성화가 필요하다 깃허브 계정에서 설정 필요하고 계정 로그인/토큰 연결 등이 필요하다
		- 그런 뒤 quartz > quartz.layout.ts 파일 수정 필요
		- 댓글창을 메인, 카테고리, 개별 글들에 선택적으로 넣을 수 있다.
		- 블로그 전체에 댓글이 연동되는지는 아직 확인을 못했다.
	- 방문자 통계
		- Google Analytics 연동
		- 다만 블로그에서 UI로 띄울 수는 없음

8. 안드로이드 git 연결
	- termux라는 어플을 사용해서 터미널 써야 쉬움. (bt 키보드가 필수다)
		- 마찬가지로 cd 로 원하는 경로 찾아간 뒤
		- git clone해주고
			- 이 과정에서 계정명과 토큰/비번을 넣어줘야 한다
		- 그 경로로 볼트를 open한다
		- obsidian git 설치한 뒤 
		- 설정에서 개인신상 4개 전부 입력해야 한다.
			- 계정명/토큰/커밋할 이름/이메일 등 넣어줘야 정상작동
	- 하지만 내 볼트가 1gb정도 되는데 status 확인하는데만 5분 넘게 소요된다. pull push도 오래걸리고.. 용량이 큰 볼트는 빠른 동기화는 기대하기 어렵다.

9. publish 파일이 볼트 밖에 있어서 깃 클론한 위치에 영향을 받았었는데
	- 볼트 내부로 publish.py 위치를 변경하고 경로를 자동으로 탐지하도록 수정
	- shell commend의 명령어는 바꿔주어야 함
		- `py D:\99_Obsidian\obsidian-vault\_config\publish.py`
